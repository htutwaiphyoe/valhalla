Folder structure

config => configuration files
controller => api routes handlers
components => react components
models => model for database
pages => web pages
pages/api => api routes
public => static files

next-connect package is a routing and middleware for nextjs and make api routes handlers easily like ExpressJS.

postman is used to test API. First create environment for project and then create collection for project and save API enpoints
in there.

mongoose is used for database. create dbConnect file in config for database connection. add environment variables in next.config.js.
In express, connection is in app.js but in nextjs, each api has to call connection method (dbConnect)

In nextjs, mongoose model needs to return existing model if it is already created. mongoose.models.ModelName || mongoose.model("Model", schema) is needed
in export default. If the condition is not included, it will return error.

seeder.js is created to add all rooms into database. To run that file, need npm script in package.json with the command
"node utils/seeder". We have to use node command to run and if we use node command, we have to change import to require.

we can retrieve json data from .json by importing it

like express, next-connect provides global error middleware which is a error-first function with 4 paramenters. If any middleware throws an error
it will execute. To throw an error, use throw or next() function. To catch all errors, give that middleware in nc({ onError }) option of next-connect
